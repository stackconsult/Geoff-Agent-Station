## 2. Conventions (Enforced by CI)

### Rust Rules
- Every Tauri command: `pub async fn` — no sync commands in `commands/` modules
- All file writes: backup-before-write via `make_bak_path()` + `fs::write(bak, content)` from in-memory string
- Failure path: restore from `.bak`, delete `.bak`, return `Err(format!())`
- Path construction: `std::path::PathBuf` only — never `format!("{}.bak", path)`
- Dead code: fix at source; scaffolding modules behind `#[cfg(feature = "...")]`
- Lint gate: `cargo clippy -- -D warnings` must be 0 errors
- No `unwrap()` on user-data paths — use `?` or explicit `map_err`

### TypeScript Rules
- `invoke` imported statically at top of file — no dynamic `await import()`
- No hardcoded filesystem paths anywhere
- Stub handlers: `toast.info('Feature coming soon')` — never `console.log('TODO')`
- Shared state: Zustand stores (Sprint I) — `useState` only for component-local UI state
- CSS: CSS custom properties via `var(--color-*)` — no inline hex values
- Tests co-located: `Component.test.tsx` next to `Component.tsx`

### Git Rules
- Commit format: `type(scope): description` — types: fix, feat, refactor, ci, test, chore
- One logical change per commit
- Gates must pass before commit: `cargo test && cargo clippy -- -D warnings && pnpm test:run && tsc --noEmit`

## 6. Per-Function Build Specs

### scan_vault (Sprint H — CRITICAL)
- **Location:** `src-tauri/src/vault/mod.rs:160-302`
- **Problem:** 200 lines, 0 tests, blocking I/O in async fn
- **Fix H1-H3:** Add 3 tests using temp dirs with controlled .md files
- **Fix H6:** Replace `std::fs::read_to_string(path)` with `tokio::fs::read_to_string(path).await`
- **Precondition:** Function signature is already `pub async fn` — only the body uses sync I/O
- **Risk:** `WalkDir` is sync-only; wrapping in `tokio::task::spawn_blocking` is the correct pattern

### VaultSelector (Sprint H)
- **Location:** `src/components/VaultSelector.tsx`
- **Test H4:** Render with no props → expects "Select vault" or similar prompt
